● ---
  analyse(string $csvPath): array — the public entry point

  This is the only public method. Everything else is private implementation detail.

  It receives the absolute path to a CSV file on disk and returns the finished statistics array. Its only job is to safely open
  the file and hand the file handle to process().

  fopen($csvPath, 'r') opens the file in read-only mode and returns a resource handle — essentially a cursor positioned at the
  start of the file. If the file does not exist, is not readable, or the path is wrong, fopen returns false instead of a handle.
  The explicit === false check catches that and throws a RuntimeException immediately, so the caller gets a clear error message
  rather than a PHP warning further down the call stack.

  Once the handle is open, process() is called inside a try block. The finally block calls fclose() unconditionally — whether
  process() returns normally, or throws an exception partway through. This guarantees the file descriptor is always released.
  Without finally, an exception thrown mid-processing would leak an open file handle.

  ---
  process($handle): array — the main loop coordinator

  This method owns the top-level structure of the algorithm: read the header, stream every data row, return the result. It
  delegates the details to the three private helpers.

  Reading the header. The first fgetcsv($handle) call consumes the very first line of the file and returns it as an array where
  each element is one column name. The file cursor advances past that line, so subsequent fgetcsv calls start from row two. If
  the file is completely empty, fgetcsv returns false — that early return produces row_count: 0 and an empty columns map rather
  than crashing.

  Cleaning the headers. array_map('trim', $headers) strips leading and trailing whitespace from every column name — common when
  CSVs are hand-edited. The second line strips a UTF-8 BOM from the first cell. A BOM is three invisible bytes (\xEF\xBB\xBF)
  that Excel silently prepends to files it saves as UTF-8. Without this strip, the first column name would contain invisible
  garbage characters that would never match anything.

  Initialising accumulators. initialise() is called once with the cleaned headers. It returns a blank slate of counters for every
   column, which accumulate() will fill in row by row.

  The streaming loop. fgetcsv is called in a while loop — it reads and parses one line per call, returning an array of cell
  values, or false at end-of-file. Because only one row is in memory at a time, the method can process arbitrarily large files
  without loading them entirely. $rowCount is incremented for each data row (the header is not counted). accumulate() is called
  with the current row on every iteration.

  Building the result. After the loop, finalise() converts the raw accumulators into the clean output structure. The method
  returns row_count (total data rows seen) and columns (one entry per header, keyed by column name).

  ---
  initialise(array $headers): array — setting up the counters

  This method creates the internal scratch space used during streaming. For each column name it creates an associative array with
   seven fields:

  - null_count — starts at zero, incremented whenever a cell is empty.
  - all_numeric — starts as true. This is a flag that represents the question "has every non-empty value we've seen so far been a
   valid number?" The moment a non-numeric value appears in this column, it flips to false and never goes back to true. Once it's
   false there is no need to check further values in that column for numeric compatibility.
  - numeric_count — the number of non-empty values that actually passed the numeric check. This is used as the denominator when
  computing the average. It is distinct from the total row count because null cells are excluded.
  - min and max — start as null so the first value encountered becomes both the min and the max unconditionally. Using null
  instead of PHP_INT_MAX / PHP_INT_MIN avoids any assumptions about the data's scale.
  - sum — a running total of all numeric values, initialised to 0.0 (a float, not an integer) to ensure float arithmetic
  throughout even if all values happen to be whole numbers.
  - value_counts — an empty associative array that will hold every distinct non-empty value mapped to how many times it appeared.
   This is used for string columns to produce the frequency map and the unique count.

  ---
  accumulate(array $headers, array $row, array &$accumulators): void — processing one row

  This is called once per data row. It iterates over the column headers by index, reads the corresponding cell from the row, and
  updates that column's accumulator. $accumulators is passed by reference so modifications are reflected in the caller's copy
  without copying the entire array on every row.

  $row[$i] ?? '' handles rows that are shorter than the header — if a row has fewer cells than expected, the missing cells are
  treated as empty strings. trim() removes whitespace and stray carriage-return characters (\r) that Windows line endings can
  leave on the last cell of a row.

  Empty cell path. If the trimmed value is an empty string, null_count is incremented and the loop moves to the next column
  immediately via continue. Nothing else is updated — an empty cell contributes nothing to frequency counts or numeric stats.

  Non-empty cell — string stats. Every non-empty value is unconditionally added to value_counts. This happens before the numeric
  checks because even a column that turns out to be numeric might later be reclassified as string (if a non-numeric value appears
   later), and by then all prior values are already in the map. The expression ($acc['value_counts'][$value] ?? 0) + 1 safely
  handles a key that does not yet exist, initialising it at zero before adding one.

  Non-empty cell — numeric gate. If all_numeric is already false for this column, the code skips the numeric checks entirely with
   continue — there is no point re-testing values for a column already known to be non-numeric. This avoids calling is_numeric()
  on every remaining value in the column after disqualification.

  Non-empty cell — numeric check. If the column still qualifies, is_numeric($value) tests whether PHP considers the string a
  valid number (integer, float, or scientific notation). If the check fails, all_numeric is set to false and the loop moves on.
  If it passes, the value is cast to a float and used to update numeric_count, sum, min, and max.

  The min/max comparison. $acc['min'] === null on the first numeric value ensures the first valid number becomes both the initial
   minimum and maximum, regardless of its actual value. After that, a simple less-than or greater-than comparison updates them.

  unset($acc) at the end of the method releases the reference variable. Without it, the $acc variable would remain bound to the
  last column's accumulator after the method returns — a PHP quirk with references in loops that can cause subtle bugs if the
  variable name is reused elsewhere.

  ---
  finalise(array $headers, array $accumulators): array — converting raw counters into output

  This method runs once after the streaming loop is complete. It iterates over the columns in header order and converts each raw
  accumulator into either a numeric stat block or a string stat block.

  The type decision. $isNumeric = $a['all_numeric'] && $a['numeric_count'] > 0 combines two conditions:
  - all_numeric must still be true — meaning no non-numeric value was ever seen in this column.
  - numeric_count must be greater than zero — meaning at least one non-empty value existed.

  The second condition handles the all-empty column case: if every cell in a column was blank, all_numeric was never set to false
   (it never had a chance to be tested), but numeric_count would still be zero. Without this guard, the code would attempt to
  divide zero by zero when computing the average. With it, an all-empty column falls through to string — a safe, descriptive
  default.

  Numeric output. min, max, and sum are taken directly from the accumulator. average is computed here rather than during
  streaming because we need the final sum and numeric_count — computing a running average during streaming would require more
  state. round(..., 4) limits the output to four decimal places. null_count tells the caller how many cells were skipped.

  String output. unique_count is count($value_counts) — the number of distinct keys in the frequency map, which is exactly the
  number of unique non-empty values seen. value_counts is the frequency map itself. null_count is the same field tracked from the
   start.

  Why numeric columns discard value_counts. By the time finalise runs, numeric columns have a fully populated value_counts map
  (it was being built during accumulate). It is simply not included in the output. The data is present in memory but not returned
   — there is no overhead from building it, and no cleanup needed.
